# 기여 가이드

이 파이프라인 저장소에 기여해 주셔서 감사합니다! 변경 사항을 제안하기 전에 아래 가이드라인을 읽어주세요.

## 브랜치 전략
- `main` : 항상 배포 가능한 안정적인 상태를 유지합니다.
- `feature/*` : 새로운 파이프라인 기능이나 변경 사항을 작업하는 브랜치입니다.
- `fix/*` : 파이프라인 버그 수정을 위한 브랜치입니다.

## 커밋 컨벤션
다음과 같은 Conventional Commits 형태를 준수해 주세요:
- `feat: 새로운 CD 스테이지 추가`
- `fix: 빌드 실패 오류 수정`
- `docs: 파이프라인 설명서 업데이트`

## Pull Request 절차
1. 변경 사항을 담은 브랜치를 생성합니다.
2. 충분한 테스트(Dry-run 또는 개발 환경 배포)를 거친 후 PR을 생성합니다.
3. 리뷰어 1명 이상의 승인(Approve)을 받아야 병합할 수 있습니다.
4. 병합 후에는 해당 변경 사항이 파이프라인에 정상적으로 반영되었는지 확인합니다.
